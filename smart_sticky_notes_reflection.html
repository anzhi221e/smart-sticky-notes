4. 安全分析

XSS via innerHTML：renderWorkspaceDropdown 和 renderWorkspaceList 将工作区名直接拼入 innerHTML。若名称含 <script> 标签将被执行。虽然数据受 RLS 保护（用户只能创建自己的工作区），仍作为防御措施修复。
修复：添加 esc() 函数——利用浏览器原生 textContent→innerHTML 转义，零依赖。所有工作区名在插入 HTML 前调用 esc(w)。
function esc(s) { const d = document.createElement('div'); d.textContent = s; return d.innerHTML; }

问题根因解决

关键洞察

  • 快速迭代 > 一次完美：标签页的 workspace filter bar 是最初核心设计，上线后发现与侧边栏重复，被移除。用户在实际使用中才能做出最准确的判断。
  • sessionStorage 作为 UI 状态层：优于 localStorage——刷新后从 config 恢复默认值，跨标签不污染。这是 PWA 中处理临时 UI 状态的最佳实践。
  • JSON in config 的灵活性：复用 smartstickynotes_config 表存储工作区列表（与 pinned_tags 一致），避免了新建表、新 RLS 策略、新迁移的整套开销。
  • 安全审查自动化：Stop hook 在提交后自动扫描新代码，在 push 前就发现了 XSS 问题。这是"左移安全"的好例子。
